# Comparison to Standard Multi-Resolution Methods

SignalForge is not the first tool to analyze signals at multiple scales.
The question is whether the scales are chosen or derived — and whether the
choice of scales is arbitrary or forced by the structure of the signal.

Note: STFT, wavelets, and EMD are all defined for time-series data. For
non-time ordered sequences — log entries, transactions, genomic positions,
ranked items — these methods do not apply. SignalForge treats any ordered
sequence as its input; the comparison below is relevant when the input
happens to be time-ordered.

---

## Short-Time Fourier Transform (STFT)

The STFT computes a spectrogram by sliding a fixed window of size `W` along
the signal and taking the Fourier transform of each segment.

**What the analyst controls**: window size `W`, hop size, window function
(Hann, Hamming, etc.). The time-frequency resolution trade-off is fixed by `W`
— wider windows give better frequency resolution, narrower windows give better
time resolution. There is no way to have both simultaneously (Heisenberg
uncertainty for signals).

**What is missing**: no relationship between measurements at different window
sizes. If you run two STFTs at `W=256` and `W=512`, the results live in
different spaces with no mathematical connection between them. There is no
lattice, no nesting, no shared coordinate system.

---

## Wavelet Transform

Wavelets address the time-frequency trade-off by using windows that scale with
frequency: narrow at high frequencies, wide at low frequencies. The dyadic
wavelet transform uses scales `{W, W/2, W/4, ...}` — powers of 2.

**What this is, in lattice terms**: the sub-lattice of `Div(H)` generated by
powers of 2. It is one chain along the prime-2 axis, ignoring all other prime
axes entirely. The choice of 2 as the scaling factor is a convention — there
is nothing mathematically privileged about it.

**What is missing**: all prime axes other than 2. A signal with structure at
3-second, 5-second, or 15-second scales has that structure distributed across
lattice positions that dyadic wavelets cannot resolve cleanly. The 3-axis
is absent.

Wavelet analysis is powerful and well-studied. For signals where dyadic
scaling captures the relevant structure, it is the right tool. The claim here
is narrower: it is a special case, not the general solution.

---

## Empirical Mode Decomposition (EMD)

EMD decomposes a signal into intrinsic mode functions (IMFs) by iteratively
extracting oscillatory components. It is adaptive — the decomposition is
data-driven, not based on a fixed basis.

**What this means**: the decomposition changes with the data. Two recordings
of the same process may produce different IMFs, making cross-recording
comparison difficult. There is no fixed coordinate system; the basis is
signal-dependent.

**What is missing**: a shared structure that makes features from different
recordings directly comparable. EMD is useful for exploratory analysis of a
single signal. It does not produce structurally invariant features in the
sense SignalForge requires.

## SignalForge

| Property | STFT | Wavelets | EMD | SignalForge |
|----------|------|----------|-----|-------------|
| Window sizes | Analyst choice | Powers of 2 | Data-driven | Divisors of H |
| Scale relationship | None | Dyadic chain | None | Full lattice |
| Cross-recording comparability | Requires matching params | Requires matching params | Difficult | Guaranteed by lattice |
| Prime axes covered | — | Prime 2 only | — | All primes dividing H |
| Boundary artifacts | Possible | Possible | N/A | None within lattice; recording boundaries handled as in any windowed method |
| Computation | O(H log H) per scale | O(H) | O(H²) typical | O(H · τ(H)), nearly linear |

The scales in SignalForge are not chosen. Given windows and a grain, the horizon
is derived as `lcm(windows + [grain])`, and the valid windows are exactly the
divisors of that horizon. The lattice structure is a consequence of unique prime
factorization. The analyst declares the windows and grain; the arithmetic
determines the measurement space.
